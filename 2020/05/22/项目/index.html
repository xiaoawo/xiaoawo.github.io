1. 首页

1.1 分类查询

  1. /categorySearch?category1Id=xxx&category2Id=xxx&分页信息

  2. controller: 调用SkuService的categorySearch(category1Id,category2Id,分页信息)

  3. service:

    • 先从缓存中查找
      缓存的key:
      缓存的value: 商品在缓存中的key
      使用的数据结构：list,LRU

      根据分页信息，计算出需要取的范围，然后从list中拿到对应的key,再根据这些key从缓存中拿到具体的数据，这个过程，如果缓存未命中，则直接跳过

    • 缓存未命中再从数据库中查找
      根据传入的数据从数据库中查找，然后写入到缓存中

  4. 返回查询结果

2. 商品详情

  1. /skuInfo?skuId=xxx

  2. controller: 调用SkuService的skuInfo(skuId)

  3. service:

    • 先从缓存中查找
      缓存的key:
      缓存的value: JSON(skuInfo)
      使用的数据结构：String

    • 缓存未命中再从数据库中查找
      先加分布式锁 

      &emsp;&emsp;加锁成功：根据传入的数据从数据库中查找，然后写入到缓存中（写入的数据中，包含一个标识，代表该数据是否有效，即是否从数据库中查询到了正确数据）  
&emsp;&emsp;加锁失败：自旋一段时间后再从缓存中读

  4. 返回查询结果
